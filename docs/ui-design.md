# UI設計

## 1. 目的

Push Link v2 の画面設計と、見た目の方針を整理するためのドキュメントです。

## 2. UI 方針

- Qiita のような情報密度の高い一覧を参考にする
- HTMX で画面全体ではなく一覧部分だけを更新する
- 左サイドバーに絞り込みと導線を集約する
- 中央カラムで縦積みフィードを見せる
- ヒーローには説明文ではなく、運用状況が分かる月次棒グラフを置く

## 3. レイアウト構成

### 3.1 トップバー

- サービス名
- 検索ボックス

検索は HTMX で `/ui/sites` を叩き、一覧だけを更新します。

### 3.2 サイドバー

- Strategy 切り替え
- フィルタ
- タグ一覧

一覧の文脈を変える操作は左側に寄せています。

### 3.3 メインフィード

- 月次登録数グラフ
- 一覧ヘッダ
- サイト一覧

カードをグリッドで並べるのではなく、1 件ずつ縦に読む前提のレイアウトにしています。

## 4. 一覧アイテムの表示要素

各サイトの一覧では以下を表示します。

- 登録日
- 登録ユーザー
- 公開状態
- タイトル
- 説明
- タグ
- 適用中 Strategy ラベル
- ドメイン
- ブックマーク数

登録日とユーザー名は横並びにし、視認性のためにアイコンを付与しています。

## 5. カラー設計

カラーは `web/static/common/colors.css` に集約しています。

- 背景色
- サーフェス色
- テキスト色
- 境界線色
- アクセント色
- 成功状態色
- シャドウ

これにより、今後テーマや共通 UI を増やす際も色定義を一箇所で管理できます。

## 6. 月次棒グラフ

ヒーローエリアには月次登録数の棒グラフを配置しています。

- 月ごとの登録数を表示
- 棒の高さで増減を直感的に表現
- 一覧説明文ではなく、運用状況を即座に把握できる情報を優先

## 7. レスポンシブ方針

- デスクトップでは `sidebar + feed` の 2 カラム
- モバイルでは 1 カラムに折りたたむ
- 上部検索と一覧操作はモバイルでも維持する
